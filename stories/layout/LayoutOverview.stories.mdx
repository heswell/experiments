# Layout

<div class="subheading">useLayout</div>

Layout management is orchestrated from the  `useLayout` hook. All layout containers 
call this hook and use the props and dispatcher it returns. The dispatcher is 
use to invoke layout actions. The props returned from useLayout replace the standard props 
passed to the component. The standard component props are enhanced by useLayout to add layout-
specific properties, both at the component level and for component children.

For the root layout only, dispatchLayoutAction is created and stored here. It is passed via 
prop drilling to all layout components in the component tree. The element tree itself is also
manipulated and stored within the hook at the root level only. 

Behaviour of `useLayout` depends on whether or not calling component is the root layout, whether 
it directly renders content or is simply a behaviour-enabling provider and whether or not JSON 
layout is provided as a prop.

- Layout container is the root container. Layout is a Provider only. e.g. DraggableLayout. JSON layout passed as prop.
- Layout container is the root container. Layout is a Provider only. e.g. DraggableLayout. No JSON layout prop.
- Layout container is the root container. Layout renders content. e.g Flexbox, Stack. JSON layout passed as prop.
- Layout container is the root container. Layout renders content. e.g Flexbox, Stack. no JSON layout prop.
- Layout container is not the root container. Layout is a Provider only.
- Layout container is not the root container. Layout renders content. e.g Flexbox, Stack.
